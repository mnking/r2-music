# R2 Music Player

A modern music player built with Next.js that streams audio files directly from your Cloudflare R2 bucket.

## Features

- **List & Search**: Browse all audio files in your R2 bucket with real-time search
- **Audio Playback**: Full-featured player with play/pause, skip, seek, and volume controls
- **Dark UI**: Sleek dark-themed interface with smooth animations
- **Secure Access**: Supports both public R2 URLs and proxied private URLs
- **Zero Server Cost**: Stream directly from R2 when using public URLs
- **Optional Supabase**: Track play history and analytics

## Tech Stack

- **Next.js 16** - React framework with App Router
- **TypeScript** - Type safety
- **Tailwind CSS** - Styling
- **shadcn/ui** - UI components
- **AWS SDK v3** - R2/S3 compatible API
- **Supabase** - Optional database for play history
- **Vercel** - Free hosting & CI/CD

## Setup

### 1. Clone & Install

```bash
cd r2-music
npm install
```

### 2. Configure Environment Variables

Copy `.env.local.example` to `.env.local`:

```bash
cp .env.local.example .env.local
```

### 3. Cloudflare R2 Setup

Get your R2 credentials from the Cloudflare dashboard:

```env
R2_ACCOUNT_ID=your_account_id
R2_ACCESS_KEY_ID=your_access_key_id
R2_SECRET_ACCESS_KEY=your_secret_access_key
R2_BUCKET_NAME=yt-music
```

**Recommended**: Enable public access and set your public URL:

```env
R2_PUBLIC_URL=https://pub-your-account-id.r2.dev
```

> **Why public URL?** Audio files stream directly from Cloudflare's edge network. This means **zero Vercel function execution time** for audio streaming, keeping you well within free tier limits even with heavy usage.

**Getting R2 Credentials:**
1. Go to [Cloudflare Dashboard](https://dash.cloudflare.com) → R2
2. Copy your **Account ID** from the right sidebar
3. Go to **Manage R2 API Tokens** → **Create API Token**
4. Select **Object Read** permissions (or **Object Read & Write**)
5. Copy the **Access Key ID** and **Secret Access Key**

**Getting Public URL:**
1. In R2, go to your bucket → **Settings**
2. Under **Public Access**, enable **R2.dev subdomain**
3. Copy the URL (format: `https://pub-xxx.r2.dev`)

### 4. Optional: Supabase Setup

Add play history tracking with Supabase:

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key
```

Create a `play_history` table:

```sql
create table play_history (
  id bigint generated by default as identity primary key,
  song_key text not null,
  song_title text not null,
  played_at timestamp with time zone default now()
);

-- Optional: Enable RLS
alter table play_history enable row level security;
create policy "Allow all" on play_history for all using (true) with check (true);
```

### 5. Run Locally

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000)

## Deploy to Vercel

### One-Click Deploy

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new)

### Via Git (Recommended for CI/CD)

1. Push code to GitHub/GitLab/Bitbucket
2. Import project on [vercel.com](https://vercel.com)
3. Add environment variables in **Project Settings** → **Environment Variables**
4. Vercel auto-deploys on every push!

### Environment Variables on Vercel

Add these in your Vercel project settings:

| Variable | Required | Description |
|----------|----------|-------------|
| `R2_ACCOUNT_ID` | Yes | Cloudflare Account ID |
| `R2_ACCESS_KEY_ID` | Yes | R2 API Token Access Key |
| `R2_SECRET_ACCESS_KEY` | Yes | R2 API Token Secret |
| `R2_BUCKET_NAME` | Yes | Your R2 bucket name |
| `R2_PUBLIC_URL` | No | Public R2 URL (recommended) |
| `NEXT_PUBLIC_SUPABASE_URL` | No | Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | No | Supabase anon/public key |

## Architecture

### Streaming Modes

| Mode | Configuration | Pros | Cons |
|------|--------------|------|------|
| **Direct (Recommended)** | Set `R2_PUBLIC_URL` | Zero Vercel costs, faster streaming | Bucket must be public |
| **Proxy** | Don't set `R2_PUBLIC_URL` | Works with private buckets | Uses Vercel function bandwidth |

### API Routes

- `GET /api/songs` - List all audio files in the bucket
- `GET /api/songs/[key]` - Stream a specific audio file (proxy mode)

## Supported Formats

MP3, WAV, FLAC, AAC, OGG, M4A, WMA

## Free Tier Limits

### Vercel (Hobby)
- **Function Execution**: 125k invocations/day
- **Bandwidth**: 100GB/day
- **Build Time**: 6,000 minutes/month

> **Tip**: Using `R2_PUBLIC_URL` means audio streaming doesn't count against Vercel function invocations or bandwidth. Only the `/api/songs` list endpoint uses functions.

### Cloudflare R2
- **Storage**: 10GB free
- **Class A Operations**: 1M/month free
- **Class B Operations**: 10M/month free
- **Egress**: Free (no egress fees!)
